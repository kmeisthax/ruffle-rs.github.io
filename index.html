Get Ruffle Player

Adobe has called it: Flash Player will reach end-of-life in 2020. To keep Flash content working and playable, install Ruffle.

Ruffle for Users

Extension install coming soon

Play legacy Flash content across the web using our player extension. Once installed, Flash content will automatically play in Ruffle instead of Adobe's legacy player. Ruffle is Free Software and will be maintained for the forseeable future, ensuring Flash content will continue to play.

For Web Developers

Ruffle for Websites

Stand-alone library coming soon

Enable your legacy Flash content on modern web browsers, including the mobile web. Installing and configuring Ruffle is easy: just include the library and it's configuration, and all static Flash content will automatically play in Ruffle. 

<script type="text/javascript">
    window.RufflePlayer = window.RufflePlayer || {};
    window.RufflePlayer.config = {
        "polyfills": ["static-content"],
        "public_path": "assets/ruffle/"
    };
</script>
<script src="assets/ruffle/ruffle.js" />

This configuration has multiple parts:

  1. We construct the player configuration object, window.RufflePlayer.config.
  2. We configure what polyfills we want to enable. In this example, we asked for static content to be replaced.
  3. We tell Ruffle where it's WASM binary is located. This must match the location where you are storing ruffle.js.
  4. We include ruffle.js, which starts the process of polyfilling Flash.

ruffle.js may also be loaded async or defer - it will run all polyfills when loaded. However, because Ruffle is also shipped as an extension, it is important that your local configuration is present before DOMContentLoaded has been fired. Ruffle for Users may run on your site, and this ensures that your local configuration is respected if the extension runs instead of your version.

Dynamic Flash content

The above configuration is suitable for websites with static Flash content and no Flash detection. However, if you want to selectively enable Ruffle for specific content, you will need to use the API.
